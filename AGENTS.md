# AGENTS.md — Jardin Sans Gaspillage

## Projet

Site Astro statique pour jardinsansgaspillage.fr.

Sujet : récupération d’eau, arrosage économe, jardin sec, cuves, récupérateurs d’eau, oyas, goutte-à-goutte, pompes solaires et solutions pour jardiner avec moins d’eau.

Objectif : site éditorial/comparatif crédible, rapide, SEO, monétisable par affiliation et vente de liens.

## Stack

- Astro statique
- CSS global simple
- JavaScript vanilla uniquement si nécessaire
- Données produits en JSON ou Markdown
- Pas de React
- Pas de Vue
- Pas de Tailwind
- Pas de Bootstrap
- Pas de librairie UI
- Pas d’icônes externes
- Pas de Google Fonts

## Design

Direction : carnet de terrain moderne.

Le site doit être sobre, naturel, éditorial, moderne, pratique et crédible.

Palette :
- encre : #1f2a24
- papier : #f7f3ea
- papier clair : #fffaf1
- sauge : #8fa58c
- sauge foncé : #4f6f52
- argile : #b76e4c
- sable : #e6d8bd
- eau discrète : #6f9fb5
- bordure : #d8cbb2

Typographie :
- titres : Georgia, Cambria, Times New Roman, serif
- texte : system-ui, sans-serif

## Interdits design

Éviter absolument :
- design IA générique
- gros gradients verts/bleus
- style SaaS
- cards répétitives partout
- icônes décoratives random
- grosses ombres molles
- boutons énormes
- fond blanc pur
- sections identiques répétées
- illustrations IA génériques

Préférer :
- fond papier
- bordures fines
- encadrés éditoriaux
- tableaux lisibles
- blocs conseil
- détails sobres
- hiérarchie claire

## SEO

Architecture cible :
- /recuperation-eau/
- /arrosage-econome/
- /jardin-sec/
- /comparatifs/
- /guides/
- /outils/

Chaque page doit avoir :
- title unique
- meta description
- H1 unique
- HTML sémantique
- maillage interne cohérent

## Affiliation

Être transparent.
Ne jamais prétendre avoir testé un produit si ce n’est pas vrai.
Expliquer les critères de comparaison : capacité, installation, robustesse, usage, prix, disponibilité, limites.

## Règles Codex

- Ne jamais refaire tout le site sans demande explicite.
- Une tâche = un objectif précis.
- Modifier le minimum de fichiers.
- Ne pas ajouter de dépendance sans nécessité.
- Garder le code simple.
- Toujours vérifier que npm run build passe quand le projet aura une base technique.
- Toujours résumer les fichiers modifiés.
